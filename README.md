## commit message要求

格式：[type] + content + file(s)

### type

1. new 新建文件或文件夹
2. delete 删除文件或文件夹
3. add 新加入内容
4. fix 修改错误或调整格式
5. final 最终定稿

### content

修改的内容

### file(s)

filename.type

### examples

[new] README.md

[delete] empty files
